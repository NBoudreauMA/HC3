What is HubbCONNECT (Agnostic Beta 2)?

A pure front-end demo of municipal workflows. In production, we wire Power Automate → OneDrive/SharePoint for governed storage, retention, and approvals. The front-end is vendor-agnostic and designed to be dropped into your municipal stack.

Modules: HubbFIX (DPW/Facilities), Licensing, AP/Turnover Lite, CIP, Budget Explorer.

Storage Envelope

{
  "version": "1.0",
  "module": "hubbfix|licensing|ap_turnover|cip",
  "ticket": "8-char slug",
  "path": "/HubbCONNECT/{module}/{year}/{date}/{ticket}/",
  "meta": { "created_at": "YYYY-MM-DDTHHMMSS" },
  "payload": { "... form data ..." },
  "files": [ { "filename": "...", "contentBase64": "...", "contentType": "..." } ]
}

Power Automate (OneDrive/SharePoint) — Reference

  1. Trigger: When an HTTP request is received (paste schema below).
  2. Create folder: root + body.path if not exists.
  3. Create file: payload_{ticket}.json with body.payload.
  4. Optional: Create file: summary_{ticket}.html based on your template.
  5. Apply to each files[]: create under attachments/.
  6. Respond 200 with {ok:true, path:<createdPath>}.

Keep your Flow URL secret. In this demo, submission is disabled.

HTTP Request Schema

{
  "type":"object",
  "properties":{
    "version":{"type":"string"},
    "module":{"type":"string"},
    "ticket":{"type":"string"},
    "path":{"type":"string"},
    "meta":{"type":"object"},
    "payload":{"type":"object"},
    "files":{"type":"array","items":{
      "type":"object",
      "properties":{"filename":{"type":"string"},"contentBase64":{"type":"string"},"contentType":{"type":"string"}},
      "required":["filename","contentBase64","contentType"]
    }}
  },
  "required":["module","ticket","path","payload"]
}
